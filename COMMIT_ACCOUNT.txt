ACCOUNT — Personal Identity & Security (Phase 1: avatar + names + sessions)
schema
- User.legalName / preferredName / legalNameProposed / legalNamePending.
- UserSession model (id == JWT `sid`, deviceInfo, ipAddress, lastSeenAt), cascade.
backend
- Login mints a UserSession; its id rides in the JWT as `sid`.
- JwtStrategy validates the session still exists (revoke -> next request 401);
  legacy tokens without a sid still accepted. Best-effort lastSeenAt heartbeat.
- account module: GET/PATCH profile, POST avatar (multer, 5 MB images),
  GET sessions, DELETE sessions/:id, DELETE sessions/others.
- Preferred name edits apply instantly; legal-name changes park in
  legalNameProposed + legalNamePending until HR/Finance clear them
  (PATCH /account/:userId/legal-name/clear, role-gated to HR/Finance/Admin).
frontend
- /account/security page: avatar upload, names (with pending-verification
  banner), active-sessions list with device/IP/last-seen + revoke / sign-out-others,
  2FA Phase-2 teaser. accountApi client.

2FA (TOTP) intentionally deferred to Phase 2.
