Live Logistics — rental command center (concept)

Turns the rental module from a list into a live operation. One map shows every on-hire caravan, generator, vehicle and equipment package — where it should be, where it actually is, who's driving, what's loaded, and which crew are on that location. Driver Approvals, Incidents and Fuel Logs become tabs because they're all field events tied to an asset + location.

Where the location data comes from (build order)

Phase 1 — Manual / booking-derived (no GPS needed): each hire gets an expected location from its booking (link a Location/shoot or type a place + dates). Drivers/coordinators tap "Arrived / Departed / Returned" in the existing driver portal. The map plots expected locations and status. This alone gives the whole command center — zero hardware.

Phase 2 — Driver portal GPS: the driver portal (already exists) adds an optional "Share location" that drops a ping on check-in or every N minutes while on a hire. Now pins show actual position, transit ETAs, and off-route/idle alerts. Privacy-respecting: only while on an active hire, only coarse pings.

Phase 3 — Telematics (optional, later): ingest GPS from a hardware tracker / fuel-card API for live continuous tracking and automatic mileage→fuel reconciliation. Same map, richer feed.
